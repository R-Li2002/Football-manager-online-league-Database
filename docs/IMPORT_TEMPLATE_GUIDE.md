# HEIGO 导入模板说明

## 1. 文档目的

这份文档只解决一件事：

- 正式导入前，Excel 和 CSV 文件应该长什么样

它面向两类维护者：

- 联赛管理员：需要准备联赛名单 Excel 和球员属性 CSV
- 开发维护者：需要排查导入失败、补齐模板、解释字段要求

当前正式导入默认走严格模式。也就是说，模板错误不会被静默容错，而是直接中止导入。

## 2. 当前导入链路

正式导入依赖两份文件：

1. 联赛 Excel
2. 球员属性 CSV

导入程序入口：

- [import_data.py](../import_data.py)

线上正式导入编排：

- [services/import_service.py](../services/import_service.py)

正式导入前会自动：

1. 解析文件
2. 严格校验模板
3. 备份当前 SQLite
4. 成功后覆盖写入

## 3. 文件与命名建议

### 3.1 推荐文件类型

- 联赛主文件：`.xlsx`
- 球员属性文件：`.csv`

### 3.2 推荐命名

系统会自动选择最新的联赛 Excel 和属性 CSV。为了减少误选，建议目录中同一时间只放一套待导入文件。

推荐命名：

- `85届初HEIGO名单v5.xlsx`
- `2600球员属性.csv`

### 3.3 推荐放置位置

本地开发：

- 项目根目录
- 或 `imports/`

Docker / 服务器部署：

- `/srv/heigo/imports/`

### 3.4 原始导入文件约定

下列 `csv / xlsx` 文件在当前项目里默认视为“导入数据库的原始数据”，而不是普通功能代码文件：

- 联赛 Excel
- 球员属性 CSV
- 按赛季或版本留存的历史导入原件

这类文件的主要用途是：

- 作为正式导入的数据来源
- 作为导入失败时的回溯与排查依据
- 作为不同届次 / 版本之间的数据留档

维护约定：

- 不要把它们当成前后端功能改动的一部分来理解
- 只有在确认“源数据本身发生变化”时，才把它们视为需要单独处理的数据更新
- 如果只是日常开发、调样式、修接口、补测试，这类文件通常不需要一起提交
- 新的导入原件可以放在项目根目录或 `imports/`，但应保持命名清晰，方便识别版本与来源

## 4. Excel 工作簿要求

联赛 Excel 至少包含以下工作表：

1. `信息总览`
2. `联赛名单`

兼容导入可额外包含：

3. `球员对应球队`

### 4.1 工作表名必须完全一致

不能写成：

- `联赛名单表`
- `信息总览1`
- `球员-对应球队`

必须完全写成：

- `信息总览`
- `联赛名单`
- `球员对应球队`

## 5. `信息总览` 工作表模板

`信息总览` 同时承担两部分内容：

1. 联赛规则区
2. 球队表

### 5.1 联赛规则区

这一块不是标准表头表格。导入程序会把前两列当成：

- 第 1 列：规则名称
- 第 2 列：规则值

常见规则项包括：

- `届数`
- `本版首届`
- `成长年龄上限`
- `超级级工资帽`
- `甲级工资帽`
- `乙级工资帽`
- `GK系数`
- 若干名额 / 身价 / PA 系数

这一块缺少部分值时，程序会回退默认值，但仍建议模板中完整保留。

### 5.2 球队表位置

程序按 `header=1` 读取球队表，也就是：

- 第 2 行会被当成球队表的表头

因此建议：

- 第 1 行保留联赛规则区
- 第 2 行开始写球队表头

### 5.3 球队表必需列

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `球队名` | 是 | 联赛内唯一球队名，后续要和 `联赛名单.联赛球队` 完全一致 | `Barcelona` |
| `级别` | 是 | 球队所属联赛级别 | `超级` |

### 5.4 球队表可选列

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `主教` | 否 | 管理或展示用途 | `R-Li` |
| `额外工资` | 否 | 额外工资或预算修正值 | `0.5` |
| `税后` | 否 | 税后标记或金额 | `是` |
| `备注` | 否 | 球队补充说明 | `已锁定阵容` |

### 5.5 球队表常见错误

1. `球队名` 重复
- 同名球队会直接报错

2. `级别` 为空
- 严格模式下会中止

3. 表头不在第 2 行
- 会导致程序识别不到列名

4. `球队名` 与 `联赛名单.联赛球队` 不一致
- 例如：
  - 这里写 `FC Bayern`
  - 联赛名单写 `Bayern`
- 严格模式会判定为未知球队

## 6. `联赛名单` 工作表模板

`联赛名单` 是正式导入的核心表。当前严格模式下，这张表的字段要求最严格。

### 6.1 必需列

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `编号` | 是 | 球员 UID，联赛内主键 | `24048100` |
| `姓名` | 是 | 球员姓名 | `Dani Olmo` |
| `年龄` | 是 | 当前年龄 | `27` |
| `当前CA` | 是 | 当前能力值 | `165` |
| `PA` | 是 | 潜力值 | `170` |
| `位置` | 是 | 位置描述，不能为空 | `M/AM/ST C` |
| `联赛球队` | 是 | 所属联赛球队，必须与 `信息总览.球队名` 完全一致 | `Barcelona` |

### 6.2 建议保留列

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `初始CA` | 否 | 初始能力值 | `159` |
| `国籍` | 否 | 公开展示用 | `Spain` |
| `俱乐部` | 否 | 现实俱乐部或来源俱乐部 | `Barcelona` |

### 6.3 兼容列名

当前导入器会兼容以下球队列名：

- `联赛球队`
- `俱乐部`
- `更新俱乐部`

但正式导入仍建议只用：

- `联赛球队`

这样模板最清楚，也更容易排查问题。

### 6.4 严格模式要求

当前正式导入默认严格模式，核心规则是：

1. `位置` 不能为空
2. `联赛球队` 不能为空
3. `联赛球队` 必须能在 `信息总览.球队名` 中找到完全一致的项
4. `编号` 不能重复

### 6.5 常见错误

1. `编号` 重复
- 同一个 UID 出现两次会直接失败

2. `位置` 为空
- 严格模式直接失败

3. `联赛球队` 为空
- 严格模式直接失败

4. 球队名不一致
- 例如：
  - `信息总览` 写 `A. Bilbao`
  - `联赛名单` 写 `Athletic Bilbao`
- 程序不会自动纠错

5. 把空白行留在表格中间
- 容易造成数据脏行

## 7. `球员对应球队` 工作表模板

这张表是兼容导入辅助表，不是正式导入的核心依赖。

当前正式导入默认严格模式，因此一般不会依赖它修正球队映射。它更适合：

- 历史数据兼容
- 临时排查旧模板

### 7.1 必需列

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `UID` | 是 | 球员 UID | `24048100` |

### 7.2 可选列

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `球队` | 否 | 兼容映射用球队名 | `Barcelona` |
| `位置` | 否 | 兼容映射用位置 | `M/AM/ST C` |

### 7.3 使用建议

- 新模板可以保留这张表，但不要依赖它修正正式导入
- 如果正式导入报球队不一致，应该先改 `联赛名单` 和 `信息总览`，而不是指望这张表兜底

## 8. 球员属性 CSV 模板

属性 CSV 是球员库和详情页的核心数据源。

### 8.1 代码层面的必需列

程序当前只强制要求这两列：

| 列名 | 是否必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `UID` | 是 | 球员 UID | `24048100` |
| `姓名` | 是 | 球员姓名 | `Dani Olmo` |

### 8.2 业务层面的真实要求

虽然代码只强制两列，但业务上不能只导这两列。

原因很简单：

- 其他缺失字段会被写成默认值
- 数值字段可能变成 `0`
- 文本字段可能变成空字符串

所以正式导入时，属性 CSV 应该始终使用完整导出版本。

### 8.4 CSV 常见错误

1. `UID` 重复
- 导入会直接报错

2. 只导出部分列
- 前台页面会出现大量 `0` 或空值

3. 列名被自动改写
- 例如 Excel 二次保存后把同名列改成 `速度.1`
- 程序对少数雷达列做了兼容，但不建议依赖这种行为

4. 编码异常
- 建议使用 UTF-8 CSV

## 10. 正式导入前检查清单

正式导入前，至少确认以下内容：

1. Excel 中存在 `信息总览` 和 `联赛名单`
2. 工作表名完全一致，没有错别字
3. `信息总览` 第 2 行是球队表表头
4. `联赛名单` 必需列完整
5. `联赛名单.位置` 没有空值
6. `联赛名单.联赛球队` 没有空值
7. `联赛名单.联赛球队` 与 `信息总览.球队名` 完全一致
8. Excel 中没有重复 UID
9. CSV 中没有重复 UID
10. 属性 CSV 是完整列版本，不是裁剪版

## 11. 导入失败时先看哪里

推荐排查顺序：

1. 维护中心正式导入弹窗
2. 正式导入摘要
3. 最近导入明细
4. 运维审计

如果需要离线排查，直接先跑 dry-run：

```powershell
D:\HEIGOOA\.venv\Scripts\python.exe D:\HEIGOOA\import_data.py `
  --workbook "D:\你的联赛Excel.xlsx" `
  --attributes-csv "D:\你的球员属性.csv" `
  --dry-run `
  --report-json "D:\HEIGOOA\strict_import_report.json"
```

## 12. 维护建议

以后如果改动以下任一内容，这份文档必须同步更新：

1. `import_data.py` 中的工作表名
2. `import_data.py` 中的列别名
3. 严格模式规则
4. 后台正式导入入口文案
5. 导入目录和部署方式
